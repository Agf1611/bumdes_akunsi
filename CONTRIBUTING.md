# Contributing

## Alur singkat
1. Buat branch dari `main`.
2. Lakukan perubahan seperlunya.
3. Uji lokal dan lint PHP.
4. Update `CHANGELOG.md` jika ada perubahan yang layak dicatat.
5. Buat pull request.

## Penamaan branch
- `fix/...` untuk perbaikan bug
- `feat/...` untuk fitur baru
- `docs/...` untuk dokumentasi
- `chore/...` untuk perapihan non-fungsional

## Sebelum merge
- pastikan tidak ada file runtime ikut ter-commit
- pastikan dokumentasi relevan ikut diperbarui
- pastikan perubahan database dicatat di `database/` dan `CHANGELOG.md`
